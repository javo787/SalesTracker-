# План и дорожная карта приложения для учёта продаж базарных торговцев

## 1. Эрезюме
Приложение предназначено для помощи базарным торговцам в учёте продаж, расходов и прибыли. Основные преимущества:
- Голосовой ввод данных на языках Центральной Азии (русский, узбекский, таджикский)
- Бесплатная интеграция с AI (Gemini Flash) для анализа продаж и предоставления советов
- Поддержка местных валют (сомони, сум, тенге, сом)
- Полностью бесплатное приложение с монетизацией через AdMob
- Офлайн-режим с синхронизацией при наличии интернета

## 2. Целевая аудитория
- Базарные торговцы в странах Центральной Азии (Таджикистан, Узбекистан, Казахстан, Киргизия)
- Мелкие розничные продавцы на рынках и ярмарках
- Предприниматели, ведущий учёт вручную или в простых таблицах
- Пользователи с ограниченным опытом использования сложных бухгалтерских систем

## 3. Уникальное торгов предложение (UTP)
1. **Голосовой ввод на местных языках** - первое в своём классе решение для ЦА
2. **Бесплатный AI-ассистент** - предоставляет советы по увеличению прибыли и анализ аномалий
3. **Поддержка множества валют ЦА** - моментальная конвертация и учёта в местных валютах
4. **Простота использования** - минимальное количество нажатий для записи продажи
5. **Полностью бесплатно** - монетизация только через ненавязчивую рекламу AdMob

## 4. Функциональные требования

### 4.1 Ядро приложения (MVP)
- Добавление товара: название, цена закупки, цена продажа
- Запись продажи в 1-2 тапа (выбор товара, количество, подтверждение)
- Автоматический расчёт прибыли: (цена продажи - цена закупки) × количество
- Учёт остатков товара на складе с уведомлениями при низком остатке
- Генерация дневных, недельных и месячных отчётов (выручка, прибыль, продано)
- Полный офлайн-режим с локальной базой данных SQLite
- Экспорт данных в Excel и PDF форматах

### 4.2 AI-интеграция (бесплатные API)
- Подключение к Gemini Flash API (бесплатный tier: 1500 запросов/день)
- AI-совет дня: персонализированные рекомендации по оптимизации ассортимента
- Обнаружение аномалий в продажах: уведомления о неожиданных падениях или всплесках
- Прогноз остатков: предсказание даты окончания товара при текущем темпе продаж
- Fallback на Groq Llama API при превышении лимита Gemini Flash

### 4.3 Голосовой ввод
- Кнопка активации голосового ввода на главном экране
- Распознавание речи нативными API Android/iOS (поддержка русского, узбекского, таджикского)
- Отправка распознанного текста в Gemini Flash для извлечения структурированных данных (тип операции, суммы, валюта, товар)
- Экран подтверждения с отображением извлечённых данных перед сохранением
- Коррекция данных в случае ошибки распознавания

### 4.4 Локализация
- Интерфейс на русском, узбекском и таджикском языках
- Переключение языка в настройках приложения
- Поддержка валют: Тоджикский сомони (TJS), Узбекский сум (UZS), Казахстанский тенге (KZT), Киргизский сом (KGS)
- Локализованные примеры товаров в онбординге (мука, помидоры, носки оптом и т.д.)
- Адаптация форматов чисел, дат и валют подLocale

### 4.5 Монетизация
- Баннерная реклама AdMob внизу экранов отчётов и статистики
- Interstitial-реклама при экспорте данных (PDF/Excel) - не более 1 раза в сессию
- Rewarded видео за расширенный AI-анализ: пользователь смотрит рекламу и получает детальный отчёт
- Запрет показа рекламы на экранах критических действий (добавление товара, запись продажи)

### 4.6 Публикация и рост
- ASO-оптимизация: описание в Google Play на трёх языках ЦА с ключевыми словами
- Привлекательные скриншоты приложения на местных языках
- Маркетинг в Telegram-каналах для предпринимателей ЦА
- Система сбора обратной связи внутри приложения
- Интеграция Firebase Analytics для отслеживания поведения пользователей

## 5. Техническая архитектура

### 5.1 Выбор платформы
- **React Native** - единый код для iOS и Android, ускоренная разработка, доступ к нативным модулям
- Альтернатива: Flutter (если команда имеет опыт)

### 5.2 Основные технологии
- **State management**: React Context или Redux Toolkit (для простоты commencer с Context)
- **Локальная база данных**: SQLite через библиотеку react-native-sqlite-storage
- **Голосовой ввод**: react-native-voice (или нативные модули для каждого платформы)
- **AI интеграция**: @google/generative-ai для Gemini Flash, groq-sdk для Groq Llama
- **Экспорт в Excel/PDF**: 
  - Excel: exceljs или react-native-xlsx
  - PDF: react-native-pdf или html-to-pdf
- **Реклама**: react-native-admob или @react-native-firebase/admob
- **Analytica**: Firebase Analytics
- **Международнаяization**: react-i18next или встроенная i18n в React Native

### 5.3 Архитектура данных
#### Сущности:
1. Товар (Product)
   - id: string
   - name: string (localized)
   - purchasePrice: number (в базовой валюте)
   - salePrice: number (в базовой валюте)
   - currency: string (TJS, UZS, KZT, KGS)
   - currentStock: number
   - minStockAlert: number (опционально)

2. Продажа (Sale)
   - id: string
   - productId: string (ссылка на товар)
   - quantity: number
   - salePricePerUnit: number (ценa в момент продажи)
   - purchasePricePerUnit: number (закупочная цена в момент продажи)
   - profit: number (рассчитывается автоматически)
   - currency: string
   - timestamp: ISO string
   - notes: string (опционально, от голосового ввода)

3. Валюта и курсы (опционально для будущих версий)
   - baseCurrency: string (например, USD для внутренних расчётов)
   - exchangeRates: { [currency]: number }

### 5.4 Архитектура голосового ввода (как в voice_flow_architecture.html)
1. Пользователь нажимает кнопку микрофона и говорит фразу
2. Speech-to-Text преобразует аудио в текст (нативные API)
3. Текст отправляется в Gemini Flash с промптом для извлечения структурированных данных
4. Приложение отображает извлечённые данные для подтверждения пользователем
5. После подтверждения данные сохраняются в SQLite
6. Обновляются статистика и проверяются остатки товаров

### 5.5 План работы с AI
- Использование бесплатного tier Gemini Flash (1500 запросов/день) достаточно для старта
- При превышении лимита автоматический переход на Groq Llama (с аналогичным бесплатным tier)
- Кеширование частых запросов для сокращения количества вызовов к API
- Оптимизация промптов для минимизации использования токенов

## 7. Метрики успеха (KPIs)

### 7.1 Продуктовые метрики
- Количество активных пользователей в день (DAU)
- Количество записанных продаж на пользователя в день
- Уровень retention на 7 и 30 день
- Количество использований голосового ввода в день
- Количество просмотров AI-советов

### 7.2 Метрики монетизации
- eCPM (эффективнаяCost per Thousandimpressions)
- Доход от рекламы на активного пользователя в день (ARPDAU)
- Процент пользователей, взаимодействующих с rewarded рекламой

### 7.3 Метрики роста
- Количество новых установок в неделю
- Коэффициент органического роста (K-factor)
- Оценка в магазинах приложений
- Количество положительных отзывов с упоминанием конкретных функций

## 8. Риски и стратегии их mitigation

| Риск | Вероятность | Влияние | Стратегия mitigation |
|------|-------------|---------|----------------------|
| Низкая активность из-за привычки к ручному учёту | Средняя | Высокое | Фокус на простоте и_speed_ записи (голосовой ввод, 1 тап), обучение через онбординг |
| Ошибки распознавания речи на местных языках | Средняя | Среднее | Возможность ручного ввода как fallback, обучение пользователей четкой дикции |
| Превышение лимита бесплатных запросов к AI API | Низкое | Среднее | Мониторинг usage, fallback на Groq, оптимизация промптов, кеширование |
| Проблемы с локализацией и культурной адаптацией | Низкое | Среднее | Вовлечение носителей языков в тестирование, консультации с локальными экспертами |
| Низкий доход от рекламы из-за низкой eCPM в регионе | Средняя | Среднее | Оптимизация размещения рекламы, тестирование разных форматов, фокус на вовлечённости |
| Технические сложности с офлайн-синхронизацией | Низкое | Высокое | Использование проверенных библиотек (SQLite), простая модель данных, тщательное тестирование |

## 9. Бюджет и ресурсы

### 9.1 Человеческие ресурсы
- 1 мобильный разработчик (React Native) - полная занятость
- 0.5 дизайнер UI/UX (частичная занятость для создания макетов и итераций)
- 0.2 бэкенд-разработчик (для настройки AI API и аналитики) - опционально
- Тестировщики: вовлечение сообщества użytkowników для бета-тестирования

### 9.2 Неденежные ресурсы
- Бесплатные tier Gemini Flash и Groq Llama API
- Бесплатные библиотеки и инструменты с открытым исходным кодом
- Google Play Console (разовая оплата $25 для публикации)
- AdMob аккаунт (бесплатный регистрация)

### 9.3 Ориентировочные сроки
- Общая продолжительность разработки: 8-10 недель
- Релиз MVP (без AI и голоса): через 3-4 недели
- Полный функционал с AI и голосом: через 6-7 недель
- Публикация в магазинах: через 8-10 недель

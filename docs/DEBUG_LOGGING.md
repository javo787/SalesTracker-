# Временное отладочное логирование (voice-sale / Gemini)

Добавлено для диагностики проблемы "Gemini не работает для голосового ввода"
(падения на Level 1/2, откат на Whisper+Groq).

## Как найти все добавленные логи

Все строки помечены комментарием `// DEBUGLOG` непосредственно над собой.
Чтобы найти все места:

```bash
grep -rn "// DEBUGLOG" backend/
```

## Где именно добавлено

- **`backend/utils/gemini.ts`** — функция `fetchGeminiWithRotation`:
  лог на каждую попытку ключа в ротации (успех/429/403/не-2xx/исключение),
  плюс финальный лог при исчерпании всех ключей. Префикс в логах: `[Gemini Rotation]`.
- **`backend/routes/voiceSale.ts`** — функция `logStep` и её вызовы на каждом
  переходе пайплайна (Level 1 → Level 2 → Level 3 → Level 4). Префикс: `[voice-sale]`.
  (Добавлено отдельным PR ранее — см. коммит с сообщением про thinkingLevel.)

## Как убрать после того, как проблема решена

1. Выполнить `grep -rn "// DEBUGLOG" backend/` — получить полный список строк.
2. Удалить помеченные строки и сам комментарий `// DEBUGLOG`.
3. Также удалить `logStep`/её вызовы в `voiceSale.ts`, если к тому моменту
   пайплайн стабилен и такая детализация больше не нужна (либо оставить —
   это не debug-специфично, а полезный постоянный мониторинг, на усмотрение).
4. Убедиться, что `[Gemini Rotation]` и `[voice-sale]` больше не встречаются
   в логах после нескольких тестовых запросов.

## Обзор

## 1. Render — Backend сервер

**Аккаунт:** parizod  
**URL сервиса:** `https://savdoapp.onrender.com`  
**План:** Free  
**Репозиторий:** `github.com/javo787/SalesTracker-` (папка `/backend`)

### Что делает
Основной backend сервер на Node.js/Express. Обрабатывает:
- Аутентификацию пользователей (Google, Telegram, Email)
- Синхронизацию данных между устройствами
- Систему ролей Владелец/Продавец (`/shop/*`)
- Push-уведомления о продажах (`/notifications/*`)
- Профили пользователей

### Почему выбран Render
- Бесплатный план включает 750 часов/месяц — достаточно для одного сервиса
- Автоматический деплой из GitHub при каждом push в `main`
- Поддержка Node.js без дополнительной настройки
- HTTPS из коробки

### Переменные окружения (Render)
| Переменная | Назначение |
|---|---|
| `JWT_SECRET` | Секретный ключ для подписи JWT токенов |
| `MONGODB_URI` | Строка подключения к MongoDB Atlas |
| `NODE_ENV` | Режим работы (production) |
| `PORT` | Порт сервера |
| `GOOGLE_CLIENT_ID` | Web Client ID для верификации Google токенов |
| `TELEGRAM_BOT_TOKEN` | Токен бота для Telegram авторизации |
| `FIREBASE_PROJECT_ID` | ID проекта Firebase (savdoapp-83d7c) |
| `FIREBASE_CLIENT_EMAIL` | Email Service Account для Firebase Admin SDK |
| `FIREBASE_PRIVATE_KEY` | Приватный ключ Service Account |
| `CRON_SECRET` | Секретный ключ для защиты endpoint уведомлений |

### Ограничения бесплатного плана
- Засыпает после 15 минут неактивности → решено через UptimeRobot
- 750 часов/месяц → хватает ровно на один сервис 24/7

## 2. MongoDB Atlas — База данных

**Аккаунт:** parizod  
**Кластер:** M0 Free Tier  
**Регион:** выбран ближайший к целевой аудитории

### Что делает
Облачная база данных MongoDB. Хранит:
- Пользователей (`users`)
- Магазины и участников (`shops`, `shopmembers`)
- Продажи (`sales`)
- Товары (`products`)

### Почему выбран MongoDB Atlas
- M0 план полностью бесплатный — 512 MB хранилища, 100 одновременных подключений
- Нативная интеграция с Mongoose (уже используется в проекте)
- Автоматические бэкапы
- 512 MB достаточно для нескольких тысяч пользователей с историей продаж

## 3. Vercel — Ads Service

**Аккаунт:** parizod  
**Проект:** savdo  
**URL:** `https://savdo-tan.vercel.app`  
**План:** Hobby (бесплатный)  
**Репозиторий:** `github.com/javo787/SalesTracker-` (папка `/ads-service`)

### Что делает
Отдельный Next.js сервис. Обрабатывает:
- Новости рынка (генерация через Tavily + Gemini)
- Объявления (classifieds)
- Оптовые предложения (wholesale)
- Admin панель для управления контентом
- Cron-задача: генерация новостей каждую ночь в 3:00

### Почему выбран Vercel
- Hobby план бесплатный для Next.js проектов
- Встроенная поддержка Cron Jobs (1 задача в день на бесплатном плане)
- Serverless функции не засыпают — каждый запрос обрабатывается независимо
- Автоматический деплой из GitHub

### Переменные окружения (Vercel)

|---|---|
| `MONGODB_URI` | Подключение к MongoDB |
| `MONGODB_DB_NAME` | Имя базы данных (savdo_ads) |
| `GEMINI_API_KEY` | Ключ Google Gemini для генерации новостей |
| `GROQ_API_KEY` | Ключ Groq как резервный AI |

## 4. UptimeRobot — Мониторинг и поддержание активности

**Аккаунт:** parizod  
**План:** Free (50 мониторов, проверка каждые 10 минут)  
**Монитор:** `https://savdoapp.onrender.com/health`

### Что делает
Каждые 10 минут отправляет HTTP GET запрос на `/health` endpoint Render сервера. Если сервер не отвечает — присылает email уведомление.

### Почему выбран UptimeRobot
Render Free план засыпает после 15 минут неактивности. Без пинга:
- Сервер засыпает → cron-job.org не может запустить уведомления в 20:00
- Первый запрос пользователя ждёт 30-60 секунд пробуждения

UptimeRobot пингует каждые 10 минут → сервер **никогда не засыпает** → cron срабатывает точно в 20:00.

### Текущий статус
- **Status:** UP ✅
- **Uptime 7 дней:** 100%
- **Uptime 30 дней:** 100%
- **Среднее время ответа:** ~110ms

## 5. cron-job.org — Планировщик уведомлений

**Аккаунт:** parizod  
**План:** Free (безлимитные задачи)  
**Задача:** Sales Reminder 20:00

### Что делает
Каждый день с понедельника по субботу в **20:00 по Душанбе (UTC+5)** отправляет HTTP POST запрос на:

```
POST https://savdoapp.onrender.com/notifications/run-reminder
Header: x-cron-secret: [секретный ключ]
```

Это запускает проверку — кто из продавцов не внёс продажи за день. Система умно определяет кого уведомлять:

- Продавец активен (4+ дней с продажами из последних 14) и не внёс данные сегодня → уведомление продавцу
- Владелец получает сводку о неактивных продавцах
- Владелец сам торгует и забыл → уведомление владельцу лично
- Новый участник (менее 3 дней) → пропускается
- Воскресенье → задача не запускается

### Расписание
```
Crontab: 0 15 * * 1,2,3,4,5,6
Часовой пояс: Asia/Tashkent (UTC+5)
Реальное время срабатывания: 20:00 по Душанбе
```

### Почему выбран cron-job.org
- Полностью бесплатный без лимитов
- Поддержка кастомных HTTP заголовков (нужно для `x-cron-secret`)
- Поддержка POST запросов с телом
- Логирование каждого запуска
- Настройка часового пояса
- Надёжнее чем `node-cron` внутри Render (который не сработает если сервер спит)

### Результат тестового запуска
```json
{ "success": true, "notified": 0, "skipped": 2 }
```
`skipped=2` — корректное поведение на старте: у участников ещё недостаточно данных об активности (нужно минимум 4 дня с продажами из 14).

## 6. Firebase — Push уведомления и аналитика

**Аккаунт:** parizod  
**Проект:** savdoapp-83d7c  
**План:** Spark (бесплатный)

### Сервисы Firebase используемые в проекте

| Сервис | Назначение | Лимит Free |
|---|---|---|
| **FCM** (Cloud Messaging) | Push уведомления на телефоны | Безлимитно |
| **Analytics** | Аналитика использования приложения | Безлимитно |
| **Messaging** (Admin SDK) | Отправка push с сервера | Безлимитно |

### Почему выбран Firebase
- FCM — стандарт де-факто для Android push уведомлений
- Уже интегрирован в проект (`@react-native-firebase/messaging`)
- Полностью бесплатный для push уведомлений при любом объёме
- Service Account позволяет отправлять уведомления с backend без ограничений

## 7. EAS (Expo Application Services) — Сборка приложения

**Аккаунт:** javocha (Expo)  
**Проект:** savdo-app  
**План:** Free

### Что делает
Облачная сборка Android APK без необходимости иметь Android Studio или мощный компьютер. EAS собирает приложение на своих серверах и присылает ссылку для скачивания.

### Профили сборки
| Профиль | Назначение | Тип файла |
|---|---|---|
| `development` | Разработка и тестирование | APK (debug) |
| `preview` | Внутреннее тестирование | APK |
| `production` | Публикация в Play Store | AAB |

### Переменные окружения (EAS)
Все `EXPO_PUBLIC_*` переменные хранятся в EAS и включаются в сборку автоматически:
- `EXPO_PUBLIC_API_URL` — URL Render backend
- `EXPO_PUBLIC_ADS_API_URL` — URL Vercel ads-service
- `EXPO_PUBLIC_GEMINI_KEY` / `_2` / `_3` — ключи Gemini AI
- `EXPO_PUBLIC_GROQ_API_KEY` — ключ Groq AI
- `EXPO_PUBLIC_GOOGLE_CLIENT_ID` — Android Client ID
- `EXPO_PUBLIC_GOOGLE_WEB_CLIENT_ID` — Web Client ID
- `EXPO_PUBLIC_TELEGRAM_BOT_USERNAME` — username Telegram бота

## Общая схема инфраструктуры

```
Телефон пользователя (Android APK)
         │
         ├── Auth / Sync / Shop
         │         ↓
         │    Render Backend
         │    (savdoapp.onrender.com)
         │         │
         │         ├── MongoDB Atlas (данные)
         │         └── Firebase Admin SDK (push)
         │
         ├── Новости / Объявления
         │         ↓
         │    Vercel Ads Service
         │    (savdo-tan.vercel.app)
         │         │
         │         └── MongoDB Atlas (контент)
         │
         └── Push уведомления
                   ↓
              Firebase FCM
                   ↑
         cron-job.org → Render (20:00 ежедневно)

UptimeRobot → ping /health каждые 10 мин → Render не спит
```
